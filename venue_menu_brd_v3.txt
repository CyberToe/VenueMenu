
BUSINESS REQUIREMENTS DOCUMENT (BRD) v3
Project: Venue Menu
Date: 2026-03-23

=====================================
1. EXECUTIVE SUMMARY
=====================================
Venue Menu is a location-based web application that enables users to discover local live music events by browsing venues and artists based on proximity, date, and time.

Primary Goal:
Enable users to discover artists playing nearby and attend events.

Key Design Principles:
- Discovery-first (no login required)
- Decentralized data creation
- Validation via show approvals only
- Media via YouTube embedding only

=====================================
2. USER TYPES
=====================================
- Public User
- Authenticated User
- Owner
- Contributor
- Platform Admin

=====================================
3. CORE USER JOURNEYS
=====================================

3.1 Public Discovery
- User lands on site
- Clicks "Find Venue"
- Inputs location/date/time
- System returns venues with top 3–5 shows
- User clicks venue or artist
- Media plays inline (YouTube embed)

3.2 Artist Creates Show (Linked)
- Select venue
- Create show
- artist_approved = true
- venue_approved = false
- Hidden until venue approves

3.3 Artist Creates Show (Unlinked)
- Enter venue name + address
- Visible immediately
- Later linking triggers approval

3.4 Venue Creates Show
- Add artists (existing or new)
- venue_approved = true
- artist_approved = false

3.5 Artist Joins Show
- Requests to join
- Venue must approve

3.6 Claim Flow
- User requests claim
- Admin approves/rejects

=====================================
5. SHOW STATE LOGIC
=====================================

Visibility Rules:
- Unlinked → visible
- Linked → requires approval
- Fully approved → visible

Creator auto-approves

Linking:
- Resets approval
- Hides show until approved

=====================================
6. API SPECIFICATION (MVP)
=====================================

Auth:
POST /api/auth/register
POST /api/auth/login

Artists:
POST /api/artists
GET /api/artists/:id
POST /api/artists/:id/media

Venues:
POST /api/venues
GET /api/venues/:id

Shows:
POST /api/shows
GET /api/shows/search
POST /api/shows/:id/approve
POST /api/shows/:id/join

Claims:
POST /api/claims
POST /api/claims/:id/approve

Search:
GET /api/search?lat=&lng=&date=

7. SEARCH & GEO
